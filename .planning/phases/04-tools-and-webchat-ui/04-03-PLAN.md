**JavaScript (inline script before closing body tag):**

Structure the script in clearly named sections:

1. **Tab switching** — `switchTab(name)` function: for each tab, toggle `border-b-2 border-indigo-500 text-indigo-400` on the button, toggle `hidden` class on the view div. When switching to 'identity' call `buildIdentityFileList()`. Views that need flex layout: add/remove `flex` class alongside `hidden`. Wire click handlers for each tab button.

2. **Message bubbles** — DOM-safe, no string injection:
   - `appendUserBubble(content)`: creates `div.flex.justify-end > div.max-w-[70%].rounded-2xl.rounded-tr-sm.px-4.py-2.bg-indigo-600.text-white.text-sm`. Sets text via `bubble.textContent = content`. Appends to `#messages`. Scrolls to bottom.
   - `appendAgentBubble()`: creates `div.flex.justify-start > div.max-w-[70%].rounded-2xl.rounded-tl-sm.px-4.py-2.bg-gray-700.text-gray-100.text-sm`. Returns the inner bubble element. Appends to `#messages`. Scrolls to bottom.
   - `appendErrorBubble(message)`: creates `div.flex.justify-start > div.max-w-[70%].rounded-2xl.px-4.py-2.bg-red-900.text-red-200.text-sm`. Sets text via `textContent`.
   - Streaming cursor: `span#stream-cursor.inline-block.w-2.h-4.bg-gray-400.animate-pulse.ml-0-5.align-middle`. Added via `appendChild`, removed via `cursor.remove()` before adding next token.

3. **Send message flow** — `sendMessage()` function:
   - Guard: if `isStreaming` return early
   - Read and trim `user-input` textarea value; if empty return
   - Parse model selector: `value.split('/')` where `[0]` is provider, `slice(1).join('/')` is model
   - Clear textarea, call `appendUserBubble`, call `appendAgentBubble`, show cursor, set `isStreaming = true`
   - Swap send/abort button visibility
   - Call `webclawHost.addMessage('user', content)` then `webclawHost.startStream({provider, model, onToken, onComplete, onError})`
   - `onToken`: remove cursor, append `token` to `agentBubble.textContent += token` — NOTE: appending to textContent is safe for text (not HTML). Re-add cursor. Scroll.
   - `onComplete`: remove cursor, reset streaming state, restore buttons
   - `onError`: remove cursor, call `appendErrorBubble(err.message)`, reset streaming state, restore buttons
   - Wire textarea keydown: `if (e.key === 'Enter' && !e.shiftKey) { e.preventDefault(); sendMessage(); }`
   - Wire send-btn click, abort-btn click (calls `webclawHost.abortStream()`)

4. **Tool event panel** — `window.addEventListener('webclaw:tool-event', handler)`:
   - Creates a card via `createElement`: `div.bg-gray-800.rounded-lg.p-2.text-xs` containing:
     - Header row: tool name in `span.font-mono.text-indigo-400`, status badge (yellow/green/red based on status string)
     - Summary paragraph: `p.text-gray-400` — text set via `textContent`
     - Expandable `details > summary + pre` for full result (text via `textContent`)
   - Insert new cards at top of `#tool-events` (insertBefore firstChild)
   - Trim to max 20 entries

5. **Settings UI** — `buildSettingsUI()`:
   - Clears `#api-keys-section` then rebuilds using `createElement` for each provider ('anthropic', 'openai', 'openrouter')
   - Each row: label + `type="password"` input + Save button + status span
   - On Save click: read input value, call `webclaw.keystore.setKey(provider, value)`, then immediately set `input.value = ''` (key must not remain in the DOM after passing to WASM), update status span
   - On mount: call `webclaw.keystore.hasKey(provider)` to show "Key saved" if already configured
   - Guard all `webclaw.*` calls with `if (window.webclaw && webclaw.keystore)`
   - Config export button: calls `webclaw.exportImport.exportConfig()`
   - Config import button: clicks hidden file input; `FileReader.readAsText` → `webclaw.exportImport.importConfig(text)`

6. **Identity Files UI** — `buildIdentityFileList()` and `loadIdentityFile(filename)`:
   - Fixed list of 6 files: `['IDENTITY.md', 'SOUL.md', 'USER.md', 'AGENTS.md', 'TOOLS.md', 'HEARTBEAT.md']`
   - Each filename: a button in `#identity-file-list`; click handler calls `loadIdentityFile(filename)` and highlights active button
   - `loadIdentityFile`: shows filename, shows Save button (unhides), sets editor to "Loading...", calls `webclaw.identity.getFile(filename)`, populates textarea via `editor.value = content`
   - Identity Save button click: calls `webclaw.identity.putFile(activeIdentityFile, editor.value)`, temporarily shows "Saving..." / "Saved"

7. **WASM ready handler** — `window.addEventListener('webclaw:host-ready', ...)`:
   - Calls `buildSettingsUI()` and `buildIdentityFileList()`
   - If `#messages` is empty, appends a welcome message

8. **Init** — call `switchTab('chat')` to set initial active state.
