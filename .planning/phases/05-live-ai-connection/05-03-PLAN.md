## Must-Haves (for verification)

1. Anthropic API call succeeds with valid key (single token test)
2. Streaming delivers tokens incrementally (not just single response)
3. Missing key shows "Enter API key in Settings" toast
4. Invalid key (401) shows "Invalid API key" toast
5. Rate limit (429) shows "Rate limited" toast
6. Model dropdown routes to correct provider (vendor/model-id format)
7. Tool calls work end-to-end with live provider
8. Provider switching updates active provider without page reload
9. Mock fallback shows clear demo mode message when no keys
10. All API keys remain encrypted at rest and only in WASM memory during use

## Testing Strategy

### Cost-Conscious Testing
- Use `claude-3-haiku-20240307` (cheapest Anthropic model)
- Use `max_tokens: 1` for validation calls
- Use short test prompts: "Say 'test'"
- Validate streaming with 5-10 token response only

### Test Checklist
- [ ] WASM loads without console errors
- [ ] Keystore opens (check IndexedDB v5)
- [ ] Keys load: console shows "Loaded provider: anthropic"
- [ ] Router registers: AvailableProviders() returns 1+ providers
- [ ] Event fires: webclaw:providers-ready received by JS
- [ ] Status shows: Green dot appears in Settings
- [ ] API works: Send message, receive streamed response
- [ ] Tools work: "Fetch example.com" invokes web_fetch
- [ ] Missing key: Clear error shown when no providers
- [ ] Invalid key: 401 error handled gracefully
- [ ] Provider switch: Change model dropdown works
- [ ] Memory safe: Keys cleared from WASM memory after registration
