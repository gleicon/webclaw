# Plan 05-02: Router Configuration with Live Keys

**Goal:** Configure provider router with decrypted API keys using RegisterProvider() for hot-swap capability.

**Success Criteria:**
- Router starts with empty config (no blocking)
- Providers are registered asynchronously as keys become available
- Hot-swap support: changing API key updates provider without reload
- Provider availability is queryable via HasProvider()

## Must-Haves (for verification)

1. Router initialized with empty keys at startup
2. Agent loop wired to router before async init
3. `loadProviderKeysAsync` calls `router.RegisterProvider()` for each valid key
4. Anthropic provider created with `NewAnthropicProvider(key)`
5. OpenAI provider created with `NewOpenAIProvider(key)`
6. OpenRouter provider created with `NewOpenRouterProvider(key, referer, title)`
7. Event `webclaw:providers-ready` dispatched with provider list
8. `HasProvider()` check before API calls in agent loop
9. Mock fallback when no providers configured
10. Clear error message in UI when provider unavailable

## Event Contract

```javascript
// Dispatched to window when providers are loaded
dispatchEvent(new CustomEvent('webclaw:providers-ready', {
  detail: {
    providers: ['anthropic', 'openai'], // available providers
    count: 2
  }
}))
```

## Dependencies

- 05-01: Async keystore loader must be implemented first
- Provider package: NewAnthropicProvider, NewOpenAIProvider, NewOpenRouterProvider must exist

## Hot-Swap Support

For future Settings tab integration:
- When user saves new API key, call `router.UnregisterProvider(name)` then `router.RegisterProvider(name, newProvider)`
- Re-dispatch `webclaw:providers-ready` event with updated list
