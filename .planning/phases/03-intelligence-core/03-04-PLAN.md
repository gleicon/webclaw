## Goal

Implement the memory storage system with hybrid vector+BM25 search. Store memories with Float32Array embeddings in IndexedDB. Support cosine similarity (0.7 weight) and BM25 keyword search (0.3 weight). Handle quota management with LRU eviction at 80% threshold.

## Requirements Addressed

- MEM-01: Store memory documents to IndexedDB with vector embedding
- MEM-02: Hybrid search with cosine (0.7) + BM25 (0.3) weighted scoring
- MEM-03: Embeddings via provider API (OpenAI text-embedding-3-small)
- MEM-05: Storage hygiene with 80% quota eviction

## Must-Haves (Goal Verification)

1. Memory stored with Float32Array embedding in IndexedDB
2. Hybrid search returns results ranked by 0.7*cosine + 0.3*BM25
3. Query embedding generated via OpenAI text-embedding-3-small
4. At 80% quota, oldest least-accessed memories are archived
5. BM25 index updated automatically when memory stored
