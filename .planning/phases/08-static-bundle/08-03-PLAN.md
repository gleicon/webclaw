## Distribution Strategy

**Primary channel: npm**
- Command: `npx webclaw-static serve`
- Benefits: No install required, automatic updates, familiar to JS developers
- Package includes pre-built dist-bundle/

**Secondary channel: GitHub Releases**
- Download: webclaw-v1.0.0.zip
- Benefits: Works without npm/Node.js, direct download
- Auto-generated on git tag push

**Tertiary channel: Docker**
- Command: `docker run -p 8080:80 gleicon/webclaw`
- Benefits: Production deployment, consistent environment

## npm Package Structure

```
webclaw-static/
├── package.json
├── bin/
│   └── webclaw-static.js      # CLI entry
├── dist-bundle/                 # Pre-built static files
│   ├── index.html
│   ├── assets/
│   └── webclaw.wasm.br
├── dist-singlefile/             # Optional: single-file builds
│   ├── webclaw.html
│   └── webclaw-ultimate.html
└── README.md
```

## CLI Commands

- `webclaw-static serve [port]` - Start static server (default port 8080)
- `webclaw-static open` - Open in browser (uses file:// or localhost)
- `webclaw-static --help` - Show usage

## GitHub Actions Workflow

Trigger: Push tag `v*.*.*`

Steps:
1. Build all bundle variants (multi-file, single-file)
2. Create GitHub Release with assets:
   - webclaw-v{version}.zip (multi-file bundle)
   - webclaw-v{version}-singlefile.zip (standard single-file)
   - webclaw-v{version}-ultimate.html (standalone)
3. Publish to npm (if NPM_TOKEN secret configured)

## User Setup Requirements

npm publishing requires:
- npm account
- NPM_TOKEN in GitHub secrets
- Package name availability: `webclaw-static`

Check availability: `npm view webclaw-static` (should return 404)

  <name>Task 2: Create CLI tool for serving static bundle</name>
  <files>bin/webclaw-static.js</files>
  <action>
    Create CLI tool that serves the static bundle:
    
    1. **Requirements:**
       - Serve static files from dist-bundle/
       - Simple, zero-dependency HTTP server
       - Default port 8080, configurable via args
       - Auto-open browser option
       - Help text
    
    2. **Implementation options:**
       - Option A: Use Node.js built-in `http` module (zero deps)
       - Option B: Use `serve` package (simpler, adds dependency)
       - **Decision:** Option A - zero dependencies align with project philosophy
    
    3. **Code structure:**
       ```javascript
       #!/usr/bin/env node
       const http = require('http');
       const fs = require('fs');
       const path = require('path');
       const { exec } = require('child_process');
       
       const DIST_DIR = path.join(__dirname, '..', 'dist-bundle');
       const PORT = process.argv.find(arg => arg.startsWith('--port='))?.split('=')[1] || 8080;
       const SHOULD_OPEN = process.argv.includes('--open') || process.argv.includes('-o');
       
       const MIME_TYPES = {
         '.html': 'text/html',
         '.js': 'text/javascript',
         '.css': 'text/css',
         '.wasm': 'application/wasm',
         '.br': 'application/brotli',
         '.json': 'application/json',
       };
       
       const server = http.createServer((req, res) => {
         let filePath = path.join(DIST_DIR, req.url === '/' ? 'index.html' : req.url);
         
         // Security: prevent directory traversal
         if (!filePath.startsWith(DIST_DIR)) {
           res.writeHead(403);
           return res.end('Forbidden');
         }
         
         // Default to index.html for SPA routing
         if (!fs.existsSync(filePath) || fs.statSync(filePath).isDirectory()) {
           filePath = path.join(DIST_DIR, 'index.html');
         }
         
         const ext = path.extname(filePath);
         const contentType = MIME_TYPES[ext] || 'application/octet-stream';
         
         fs.readFile(filePath, (err, content) => {
           if (err) {
             res.writeHead(404);
             return res.end('Not found');
           }
           
           // Set compression headers for .br files
           if (ext === '.br' || filePath.endsWith('.wasm.br')) {
             res.setHeader('Content-Encoding', 'br');
             res.setHeader('Content-Type', 'application/wasm');
           } else {
             res.setHeader('Content-Type', contentType);
           }
           
           res.writeHead(200);
           res.end(content);
         });
       });
       
       server.listen(PORT, () => {
         console.log(`WebClaw serving at http://localhost:${PORT}`);
         console.log(`Press Ctrl+C to stop`);
         
         if (SHOULD_OPEN) {
           const url = `http://localhost:${PORT}`;
           const command = process.platform === 'darwin' ? `open ${url}` :
                          process.platform === 'win32' ? `start ${url}` :
                          `xdg-open ${url}`;
           exec(command);
         }
       });
       ```

       ```yaml
       name: Release
       
       on:
         push:
           tags:
             - 'v*.*.*'
         workflow_dispatch:
       
       jobs:
         build:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v4
             
             - name: Setup Node.js
               uses: actions/setup-node@v4
               with:
                 node-version: '20'
                 registry-url: 'https://registry.npmjs.org'
             
             - name: Install dependencies
               run: npm ci
             
             - name: Build all bundles
               run: npm run build:all
             
             - name: Create release archives
               run: |
                 mkdir -p release
                 # Multi-file bundle
                 cd dist-bundle && zip -r ../release/webclaw-${GITHUB_REF#refs/tags/}.zip . && cd ..
                 # Single-file bundle  
                 cd dist-singlefile && zip -r ../release/webclaw-${GITHUB_REF#refs/tags/}-singlefile.zip . && cd ..
                 # Ultimate standalone
                 cp dist-singlefile/webclaw-ultimate.html release/webclaw-${GITHUB_REF#refs/tags/}-ultimate.html
             
             - name: Upload artifacts
               uses: actions/upload-artifact@v4
               with:
                 name: release-bundles
                 path: release/
         
         release:
           needs: build
           runs-on: ubuntu-latest
           permissions:
             contents: write
           steps:
             - name: Download artifacts
               uses: actions/download-artifact@v4
               with:
                 name: release-bundles
                 path: release/
             
             - name: Create GitHub Release
               uses: softprops/action-gh-release@v1
               with:
                 files: release/*
                 generate_release_notes: true
               env:
                 GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
         
         publish:
           needs: build
           runs-on: ubuntu-latest
           if: github.event_name == 'push' && startsWith(github.ref, 'refs/tags/v')
           steps:
             - uses: actions/checkout@v4
             
             - name: Setup Node.js
               uses: actions/setup-node@v4
               with:
                 node-version: '20'
                 registry-url: 'https://registry.npmjs.org'
             
             - name: Install dependencies
               run: npm ci
             
             - name: Build
               run: npm run build:all
             
             - name: Publish to npm
               run: npm publish --access public
               env:
                 NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
               continue-on-error: true  # Don't fail release if npm publish fails
       ```

  <name>Task 4: Create Dockerfile for containerized serving</name>
  <files>Dockerfile, .dockerignore</files>
  <action>
    Create Docker distribution for production deployment:
    
    1. **Requirements:**
       - Minimal image size (use nginx:alpine)
       - Serve static files efficiently
       - Support brotli compression via nginx
       - Easy to run: `docker run -p 8080:80 gleicon/webclaw`
    
    2. **Dockerfile:**
       ```dockerfile
       # Multi-stage build for minimal size
       FROM node:20-alpine AS builder
       
       WORKDIR /app
       
       # Copy package files
       COPY package*.json ./
       RUN npm ci
       
       # Copy source and build
       COPY . .
       RUN npm run build:all
       
       # Production stage
       FROM nginx:alpine
       
       # Copy built static files
       COPY --from=builder /app/dist-bundle /usr/share/nginx/html
       
       # Configure nginx for brotli and SPA routing
       RUN echo 'server { \
           listen 80; \
           server_name localhost; \
           root /usr/share/nginx/html; \
           index index.html; \
           \
           # Brotli compression support \
           location ~ \\.wasm\\.br$ { \
               add_header Content-Encoding br; \
               add_header Content-Type application/wasm; \
           } \
           \
           # SPA routing - serve index.html for all routes \
           location / { \
               try_files $uri $uri/ /index.html; \
           } \
       }' > /etc/nginx/conf.d/default.conf
       
       EXPOSE 80
       
       CMD ["nginx", "-g", "daemon off;"]
       ```
    
    3. **.dockerignore:**
       ```
       node_modules
       .git
       .github
       test/
       tests/
       *.md
       .env*
       dist-bundle/
       dist-singlefile/
       ```
