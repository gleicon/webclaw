## Gap Analysis

**Current State:**

- webclaw-ultimate.html: 2.7MB (2726KB)
- Target size: ~1.3MB (1300KB)
- Variance: +108% over target

**Root Cause Hypothesis:**
The build process likely:

1. Takes webclaw.wasm.br (Brotli compressed, ~885KB)
2. Inlines it as base64 (adds ~33% overhead = ~1177KB)
3. PLUS embeds a separate copy of the decompressed WASM (~3MB)
4. Result: ~2.7MB total

**Required Fix:**

- Only inline the .wasm.br file (compressed)
- Add client-side Brotli decompression using a JS library
- OR switch to gzip compression with native browser decompression

    Fix the ultimate bundle size issue. Two possible approaches:
    
    **Approach A: Use gzip + native decompression (RECOMMENDED)**
    - Brotli (.br) requires external library
    - gzip can use native DecompressionStream API in browsers
    - Trade-off: gzip is ~15% larger than Brotli, but no library needed
    
    **Approach B: Only inline .wasm.br with Brotli library**
    - Keep using .br file
    - Add small Brotli decompression library (inflate only, ~10KB)
    - Better compression, slightly more complex
    
    **Implementation for Approach A (native gzip):**
    
    1. Modify inline-wasm.js to:
       ```javascript
       import fs from 'fs';
       import zlib from 'zlib';
       
       export function inlineWASM(html, wasmPath) {
         // Read the Brotli compressed WASM
         const compressedWasm = fs.readFileSync(wasmPath);
         
         // Re-compress with gzip for native browser support
         const gzipWasm = zlib.gzipSync(compressedWasm, { level: 9 });
         
         // Base64 encode
         const base64Wasm = gzipWasm.toString('base64');
         
         // Create inline decompressor using native DecompressionStream
         const decompressor = `
           async function loadWasm() {
             const base64 = "${base64Wasm}";
             const compressed = Uint8Array.from(atob(base64), c => c.charCodeAt(0));
             const stream = new Response(compressed).body.pipeThrough(new DecompressionStream('gzip'));
             const decompressed = await new Response(stream).arrayBuffer();
             return decompressed;
           }
         `;
         
         // Inject into HTML
         return html.replace('/* WASM_PLACEHOLDER */', decompressor);
       }
       ```
    
    2. Update build-singlefile.js if needed to pass correct path
    
    3. Alternative: Just use raw .wasm.br and inline a small Brotli decompressor:
       ```javascript
       // Use Brotli decompression library (just the decompressor)
       import { decompress } from 'brotli-wasm'; // Only import decompress
       ```
    
    4. **Simplest fix**: Check if we're inlining BOTH files and remove duplication
    
    **Most likely fix needed:**
    The 2.7MB suggests we're inlining ~2MB of decompressed WASM PLUS the compressed version.
    
    Check build-singlefile.js around lines 70-80:
    ```javascript
    if (inlineWASMFlag) {
      const wasmPath = path.join(WASM_DIR, "webclaw.wasm.br");

    }
    ```
    
    The issue may be that even in ultimate mode, copyWASM() is also being called,
