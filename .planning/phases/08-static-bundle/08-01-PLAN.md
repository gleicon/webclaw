## Current Structure

Current index.html loads:
- CDN Tailwind CSS (external dependency)
- static/wasm_exec.js (17KB, unbundled)
- static/webclaw-host.js (11KB, unbundled)
- worker.js loads via new Worker() (8.7KB, separate fetch)
- WASM via fetch() to dist/webclaw.wasm (4.2MB raw, 865KB brotli)

## Required Vite Features

1. **Copy WASM files:** Vite doesn't handle .wasm by default, need copy plugin
2. **Handle Web Workers:** worker.js needs to be bundled separately
3. **Process Tailwind:** Replace CDN with compiled CSS
4. **Brotli compression:** Keep .br files for optimal size
5. **Relative paths:** All assets use relative paths for file:// compatibility
</context>

<task type="auto">
  <name>Task 1: Create Vite configuration</name>
  <files>vite.config.js</files>
  <action>
    Create vite.config.js with the following configuration:
    
    1. **Base config:**
       - base: './' (relative paths for file:// compatibility)
       - build.target: 'es2020' (broad browser support)
       - build.outDir: 'dist-bundle' (separate from existing dist/)
    
    2. **Entry points:**
       - main: index.html
       - worker: static/worker.js (as separate chunk)
    
    3. **WASM handling:**
       - Use vite-plugin-static-copy to copy dist/webclaw.wasm and dist/webclaw.wasm.br to output
       - Configure proper MIME type headers for .wasm and .wasm.br
    
    4. **Build optimization:**
       - rollupOptions: manualChunks for code splitting
       - Keep worker.js as separate file (required for new Worker())
       - Minification: terser for JS, esbuild target
       - CSS: extract to single file, minify
    
    5. **Development server:**
       - proxy settings if needed
       - headers for COOP/COEP (required for SharedArrayBuffer if used)
    
    Configuration structure:
    ```javascript
    import { defineConfig } from 'vite';
    import { viteStaticCopy } from 'vite-plugin-static-copy';
    import { resolve } from 'path';
    
    export default defineConfig({
      base: './',
      build: {
        target: 'es2020',
        outDir: 'dist-bundle',
        assetsDir: 'assets',
        rollupOptions: {
          input: {
            main: resolve(__dirname, 'index.html'),
          },
          output: {
            manualChunks: {
              'wasm-exec': ['./static/wasm_exec.js'],
            },
          },
        },
        // Copy WASM files separately
      },
      plugins: [
        viteStaticCopy({
          targets: [
            {
              src: 'dist/webclaw.wasm',
              dest: ''
            },
            {
              src: 'dist/webclaw.wasm.br',
              dest: ''
            },
            {
              src: 'static/worker.js',
              dest: ''
            }
          ]
        })
      ]
    });
    ```
    
    Important: worker.js must remain as a separate file (not bundled into main chunk) because it's loaded via `new Worker('static/worker.js')`. Configure rollup to treat it as an external or use ?worker import.
    
    Alternative approach: Use `import Worker from './static/worker.js?worker'` pattern and update webclaw-host.js to use the bundled worker.
  </action>

  <name>Task 4: Create entry point module for bundling</name>
  <files>src/main.js, src/worker-entry.js</files>
  <action>
    Create ES module entry points for Vite to bundle:
    
    1. **src/main.js** (main thread entry):
       - Import '../static/wasm_exec.js' (makes it part of bundle)
       - Import '../static/webclaw-host.js' (or refactor to ES module)
       - Import '../src/styles.css' (Tailwind compilation trigger)
       - Any initialization code
       - Export nothing (side-effect entry)
    
    2. **src/worker-entry.js** (Web Worker entry):
       - Import '../static/worker.js' content as ES module
       - This allows Vite to bundle and process worker.js
       - Use in index.html: `new Worker(new URL('./src/worker-entry.js', import.meta.url), {type: 'module'})`
    
    3. **Update index.html:**
       - Replace script tags with module imports:
         ```html
         <script type="module" src="./src/main.js"></script>
         ```
       - Remove: `<script src="static/wasm_exec.js"></script>`
       - Remove: `<script src="static/webclaw-host.js"></script>`
    
    4. **Handle worker loading:**
       - Option A: Use Vite's `?worker` suffix: `import Worker from './src/worker-entry.js?worker'`
       - Option B: Keep worker.js as static file, copy to dist
       - Recommended: Option A for better optimization
    
    Implementation approach:
    ```javascript
    // src/main.js
    import '../static/wasm_exec.js';
    import '../static/webclaw-host.js';
    import './styles.css';
    
    // Module-side-effects only - webclaw-host auto-initializes
    console.log('[main] WebClaw bundle loaded');
    ```
    
    Note: wasm_exec.js and webclaw-host.js use IIFE pattern currently. They may need wrapping or the vite.config.js needs to handle IIFE modules properly via `format: 'iife'` output option.
