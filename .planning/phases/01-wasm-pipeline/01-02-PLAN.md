Host page loading pattern (from official Go wasm_exec.html, adapted):
```javascript
// static/webclaw-host.js
(async function() {
    const go = new Go();  // Go class provided by wasm_exec.js

    // Polyfill for browsers without instantiateStreaming
    if (!WebAssembly.instantiateStreaming) {
        WebAssembly.instantiateStreaming = async (resp, importObject) => {
            const source = await (await resp).arrayBuffer();
            return await WebAssembly.instantiate(source, importObject);
        };
    }

    // Wait for bridges before exposing smoke test
    window.addEventListener('webclaw:ready', () => {
        console.log('webclaw: bridges available', window.webclaw);
        // Smoke test instructions logged to console:
        console.log('Smoke test:\n  window.webclaw.jsFetch("https://example.com").then(r => r.text()).then(console.log)\n  window.webclaw.jsIndexedDB.open("test-db", 1)');
    }, { once: true });

    try {
        const result = await WebAssembly.instantiateStreaming(
            fetch("dist/webclaw.wasm"),
            go.importObject
        );
        go.run(result.instance);  // starts Go runtime; main() runs asynchronously
    } catch (err) {
        console.error("webclaw: failed to load WASM:", err);
    }
})();
```

Dev server (sets correct headers for .wasm.br files):
```go
// cmd/devserver/main.go — standard Go binary, NOT compiled for WASM
package main

import (
    "log"
    "net/http"
    "strings"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        if strings.HasSuffix(r.URL.Path, ".wasm.br") {
            w.Header().Set("Content-Type", "application/wasm")
            w.Header().Set("Content-Encoding", "br")
            w.Header().Set("Vary", "Accept-Encoding")
        }
        http.FileServer(http.Dir(".")).ServeHTTP(w, r)
    })
    log.Println("Serving on http://localhost:8080")
    log.Fatal(http.ListenAndServe(":8080", mux))
}
```

Makefile pattern:
```makefile
GOROOT := $(shell go env GOROOT)
WASM_EXEC_JS := $(GOROOT)/lib/wasm/wasm_exec.js

.PHONY: build serve clean

build:
	mkdir -p dist static
	GOOS=js GOARCH=wasm go build -o dist/webclaw.wasm ./cmd/webclaw/
	brotli --best -f dist/webclaw.wasm -o dist/webclaw.wasm.br
	cp $(WASM_EXEC_JS) static/wasm_exec.js

serve:
	go run ./cmd/devserver/

clean:
	rm -f dist/webclaw.wasm dist/webclaw.wasm.br static/wasm_exec.js
```

Critical: NEVER hardcode a GOROOT path. wasm_exec.js moved from misc/wasm/ to lib/wasm/ in Go 1.24. Always use $(shell go env GOROOT)/lib/wasm/wasm_exec.js.

Critical: webclaw-host.js loads dist/webclaw.wasm (uncompressed). The server transparently handles content-encoding. Do NOT load dist/webclaw.wasm.br directly in JS — let the server negotiate encoding.

Note: wasm_exec.js must be loaded BEFORE webclaw-host.js (it provides the `Go` class).

**static/webclaw-host.js** — implement exactly as shown in the interfaces block. The file must:
1. Create `const go = new Go()` (Go class from wasm_exec.js)
2. Include the instantiateStreaming polyfill for older browsers
3. Add a `window.addEventListener('webclaw:ready', ...)` listener BEFORE calling `go.run()` — the event fires from Go's main() via jsbridge.Init(), which runs asynchronously after go.run() starts. If the listener is added after go.run(), it may miss the event.
4. Call `WebAssembly.instantiateStreaming(fetch("dist/webclaw.wasm"), go.importObject)` and then `go.run(result.instance)`
5. Log smoke test instructions to console from inside the webclaw:ready handler

**Makefile** — implement as shown in the interfaces block. Targets: `build`, `serve`, `clean`. The build target must:
- Create dist/ and static/ dirs with `mkdir -p`
- Set `GOOS=js GOARCH=wasm go build -o dist/webclaw.wasm ./cmd/webclaw/`
- Run `brotli --best -f dist/webclaw.wasm -o dist/webclaw.wasm.br`
- Copy wasm_exec.js using the dynamic GOROOT path: `cp $(GOROOT)/lib/wasm/wasm_exec.js static/wasm_exec.js`
- All on separate lines with tab indentation (Makefile requirement)

**cmd/devserver/main.go** — standard Go server (no WASM build tag — this binary runs on the host OS, not in the browser). Implement as shown in the interfaces block. No external dependencies. Must set Content-Type + Content-Encoding + Vary headers for `.wasm.br` files. Serves from the project root directory.
