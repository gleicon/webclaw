## What Was Built

### OAUTH_PROVIDERS Extension

All four providers now have authType field (oauth or pat). PAT providers (GitHub, Notion) have patScopes, patLink, patLinkText, and optionally patNote. OAuth providers retain their existing instructions/link/linkText structure.

### New Helper Functions

- setStatusDot(statusSpan, color, text, textClass) — replaces status dot rendering with safe DOM methods. Creates inline-flex span with colored dot + text label.
- showMaskedState(providerId, statusSpan, patSection, patInputRow, updateBtn) — transitions PAT card to masked state after successful save.
- buildOAuthCard(provider, container, storedClientIds) — builds Twitter/Google cards with auth-type badge (OAuth 2.0), permanently-visible redirect URI callout with copy button, client ID config row, connect/disconnect button.
- buildPATCard(provider, container) — builds GitHub/Notion cards with auth-type badge (Access Token), password-type PAT input, show/hide toggle, save button, masked state container with Update token link, scopes hint (GitHub), page-sharing callout (Notion), token creation link.

### Updated updateAllConnectionStatus()

Now handles three connected states:
1. status.valid === false — red dot "Invalid token", shows patInputRow and updateBtn for re-entry
2. status.auth_type === "pat" — shows masked state, green dot "Connected"
3. OAuth connected — green dot "Connected (username)", updates disconnect button

## Deviations from Plan

### Auto-fixed Issues

1. [Rule 1 - Bug] Replaced status dot rendering with safe DOM helper (setStatusDot)
   - Found during: Task 1 implementation
   - Issue: Project security hook blocked Edit tool calls containing status span updates using direct string assignment. The plan's status dot pattern triggered the hook.
   - Fix: Introduced setStatusDot() helper using createElement + createTextNode + appendChild throughout. Eliminates every dynamic string-based DOM assignment in Connected Services.
   - Files modified: index.html
   - Commit: ad6e4c4

2. [Rule 1 - Bug] Replaced disconnect modal template with DOM methods
   - Found during: Task 1 implementation — same trigger as above
   - Issue: Existing disconnect modal used template literal assignment. Required replacement with createElement for each element.
   - Fix: Built modal entirely with createElement + textContent.
   - Files modified: index.html
   - Commit: ad6e4c4

3. [Rule 1 - Bug] Fixed WASM export test timing in 02-pat-save-flow.spec.mjs
   - Found during: Task 2 test run
   - Issue: Export-existence tests waited for window.webclaw.oauth (set by JS at startup) then immediately checked for savePATToken. WASM goroutine registers exports 300ms+ after webclaw.oauth is initialized, causing false negatives.
   - Fix: Updated waitForFunction predicates to wait for typeof window.webclaw.oauth.savePATToken === 'function' directly.
   - Files modified: test/phase09.1-e2e/02-pat-save-flow.spec.mjs
   - Commit: c30ac25

4. [Rule 1 - Bug] Optimistic UI mask before async savePATToken
   - Found during: Checkpoint verification by orchestrator
   - Issue: showMaskedState() was called inside the try block after await savePATToken, so DOM updated only after async completion. Playwright assertions checking masked state immediately after Save click saw unmasked state.
   - Fix: Moved showMaskedState() call synchronously before the await. Added revert logic in catch block to restore input row if save fails.
   - Files modified: index.html
   - Commit: 748192f
