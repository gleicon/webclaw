## Decisions Made

- AuthType omitempty ensures backward compatibility — no IndexedDB version bump needed because AuthType lives inside the encrypted ciphertext blob, not in the schema
- Positive check `token.AuthType == "pat"` used everywhere — avoids future regressions if new auth types are added
- invalidTokens is a transient in-memory map (not persisted) — correct behavior: 401/403 marks invalid, page reload clears all invalid flags and user updates token via UI
- savePATToken async, markInvalid sync — matches whether the operation touches IndexedDB
