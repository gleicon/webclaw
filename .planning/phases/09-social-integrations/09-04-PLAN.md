<objective>
Implement GitHub integration with tools for managing issues, pull requests, code search, and comments using OAuth-authenticated API calls (REST v3 with optional GraphQL v4).
</objective>

<context>
GitHub integration is Priority 1 for developer-focused productivity (09-RESEARCH.md). Users want to manage development workflow through natural language like "What issues are assigned to me?" or "Show open PRs in the webclaw repo."

**API Details:**
- REST API v3: https://api.github.com
- GraphQL API v4: https://api.github.com/graphql (optional, for complex queries)
- Authentication: Bearer token from OAuth 2.0
- Rate limits: 5,000 requests/hour for OAuth apps
- Scopes needed: repo, issues, pull_requests, read:user

**REST API v3 Endpoints:**
- GET /user/issues - List issues assigned to authenticated user
- GET /repos/{owner}/{repo}/issues - List repo issues
- POST /repos/{owner}/{repo}/issues - Create issue
- GET /repos/{owner}/{repo}/pulls - List pull requests
- GET /repos/{owner}/{repo}/pulls/{number} - Get PR details
- POST /repos/{owner}/{repo}/issues/{number}/comments - Comment on issue/PR
- GET /search/code - Search code across repositories
- GET /user/repos - List user repositories

**GitHub Search Syntax:**
- repo:owner/name - limit to repository
- language:go - filter by language
- path:internal/ - search in path
- extension:go - file extension
- "search term" - exact phrase
- TODO - search for literal

**Tool Pattern:**
Follow WebClaw conventions. Many GitHub operations require owner/repo context - tools should accept these as params or use smart defaults (e.g., current repo from context if available).

**Deferred (not in this plan):**
- GraphQL API (v4) - implement REST first, GraphQL as enhancement
- Creating PRs - complex (need branch, commits, push)
- Merging PRs - requires merge strategy, tests, etc.
- Repository creation - less common, can defer
- Actions/workflow management - niche use case
- Gists - separate integration if needed

Reference: 09-CONTEXT.md Section "GitHub Use Cases"
</context>

  <files>internal/integrations/github/types.go</files>
  <action>
    Define Go structs for GitHub API v3 responses.
    
    1. Issue structure:
       ```go
       type Issue struct {
           ID        int64  `json:"id"`
           Number    int    `json:"number"`
           Title     string `json:"title"`
           Body      string `json:"body"`
           State     string `json:"state"` // open, closed
           User      *User  `json:"user"`
           Assignees []*User `json:"assignees"`
           Labels    []*Label `json:"labels"`
           CreatedAt string `json:"created_at"` // ISO 8601
           UpdatedAt string `json:"updated_at"`
           HTMLURL   string `json:"html_url"` // Web URL
           Comments  int    `json:"comments"`
       }
       ```
    
    2. Pull Request structure (extends Issue with PR-specific fields):
       ```go
       type PullRequest struct {
           ID        int64  `json:"id"`
           Number    int    `json:"number"`
           Title     string `json:"title"`
           Body      string `json:"body"`
           State     string `json:"state"`
           User      *User  `json:"user"`
           Head      *PRBranch `json:"head"` // Source branch
           Base      *PRBranch `json:"base"` // Target branch
           Draft     bool   `json:"draft"`
           CreatedAt string `json:"created_at"`
           UpdatedAt string `json:"updated_at"`
           HTMLURL   string `json:"html_url"`
           Comments  int    `json:"comments"`
       }
       
       type PRBranch struct {
           Ref  string     `json:"ref"`  // Branch name
           Sha  string     `json:"sha"`
           Repo *Repository `json:"repo"`
       }
       ```
    
    3. Repository structure:
       ```go
       type Repository struct {
           ID          int64  `json:"id"`
           Name        string `json:"name"`
           FullName    string `json:"full_name"` // owner/repo
           Description string `json:"description"`
           Private     bool   `json:"private"`
           HTMLURL     string `json:"html_url"`
           Owner       *User  `json:"owner"`
           OpenIssuesCount  int `json:"open_issues_count"`
       }
       ```
    
    4. User and supporting types:
       ```go
       type User struct {
           ID        int64  `json:"id"`
           Login     string `json:"login"` // Username
           HTMLURL   string `json:"html_url"`
           Type      string `json:"type"` // User, Organization
       }
       
       type Label struct {
           Name        string `json:"name"`
           Color       string `json:"color"`
           Description string `json:"description"`
       }
       
       type Comment struct {
           ID        int64  `json:"id"`
           Body      string `json:"body"`
           User      *User  `json:"user"`
           CreatedAt string `json:"created_at"`
           HTMLURL   string `json:"html_url"`
       }
       ```
    
    5. Search response:
       ```go
       type CodeSearchResult struct {
           TotalCount int `json:"total_count"`
           Items      []*CodeSearchItem `json:"items"`
       }
       
       type CodeSearchItem struct {
           Name       string     `json:"name"`
           Path       string     `json:"path"`
           Sha        string     `json:"sha"`
           URL        string     `json:"url"`
           HTMLURL    string     `json:"html_url"`
           Repository *Repository `json:"repository"`
           TextMatches []TextMatch `json:"text_matches"`
       }
       
       type TextMatch struct {
           Fragment string `json:"fragment"`
           Matches  []Match `json:"matches"`
       }
       ```
       
    6. Error response:
       ```go
       type GitHubError struct {
           Message string `json:"message"`
           DocumentationURL string `json:"documentation_url"`
           Errors []ErrorDetail `json:"errors,omitempty"`
       }
       ```

<task type="auto">
  <name>Task 2: Build GitHub API client</name>
  <files>internal/integrations/github/client.go</files>
  <action>
    Create authenticated GitHub API client with REST v3 support.
    
    1. Define Client struct:
       ```go
       type Client struct {
           baseURL    string // https://api.github.com
           oauthMgr   *oauth.Manager
           rateLimit  *RateLimit // Track rate limit from headers
       }
       
       type RateLimit struct {
           Limit     int
           Remaining int
           Reset     time.Time
       }
       
       func NewClient(oauthMgr *oauth.Manager) *Client
       ```
    
    2. Implement core methods:
       - `GetIssues(state string, assignee string, labels []string) ([]*Issue, error)`
         - GET /user/issues (issues across all repos for user)
         - Query params: state, assignee, labels, sort, direction
         - Or GET /repos/{owner}/{repo}/issues for specific repo
       
       - `GetPullRequests(owner, repo, state string) ([]*PullRequest, error)`
         - GET /repos/{owner}/{repo}/pulls
         - Query: state (open, closed, all), sort, direction
       
       - `CreateIssue(owner, repo, title, body string, labels []string) (*Issue, error)`
         - POST /repos/{owner}/{repo}/issues
         - Body: {title, body, labels}
       
       - `GetRepository(owner, repo string) (*Repository, error)`
         - GET /repos/{owner}/{repo}
       
       - `SearchCode(query string) (*CodeSearchResult, error)`
         - GET /search/code
         - Query: q (search string)
         - GitHub search has strict rate limits (10 req/min)
       
       - `CreateComment(owner, repo string, issueNumber int, body string) (*Comment, error)`
         - POST /repos/{owner}/{repo}/issues/{number}/comments
         - Works for both issues and PRs (PRs are issues under the hood)
    
    3. Implement pagination:
       - GitHub uses Link header for pagination
       - Parse rel="next" links
       - Helper to fetch all pages for small result sets
       
    4. Authentication and headers:
       - Authorization: Bearer {token}
       - Accept: application/vnd.github.v3+json (API version)
       - User-Agent: webclaw/1.0
    
    5. Rate limit tracking:
       - Parse X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset headers
       - Store in client for introspection
       - Return error if rate limited with time until reset
       
    Use jsbridge.Fetch for all HTTP - no net/http.
  </action>
