<objective>
Build OAuth 2.0 PKCE infrastructure with popup-based flow, encrypted IndexedDB token storage, and automatic refresh handling for WebClaw browser integrations.
</objective>

<context>
WebClaw requires OAuth-based integrations that work entirely in the browser. Standard OAuth with client secrets is insecure for browser apps, so we implement PKCE (Proof Key for Code Exchange) which is the industry standard for SPAs.

**Key Technical Constraints:**
- No server to handle OAuth redirects (WebClaw is a static file)
- Must use popup window flow with postMessage callback
- Tokens must be encrypted using Web Crypto API (same as API keys in Phase 2)
- All providers require PKCE: Twitter/X, Google, GitHub, Notion
- Token refresh must be automatic and invisible to user

**Integration Pattern:**
OAuth Manager is shared infrastructure used by all integration tools (Twitter, Google, GitHub, Notion). Each tool calls `oauthManager.GetToken(provider)` at execution time, which returns a valid token (auto-refreshing if needed).

**Security Model:**
- Tokens stored in IndexedDB with AES-256-GCM encryption
- Same passphrase used for API keys also encrypts OAuth tokens
- Access tokens short-lived (1-2 hours), refresh tokens long-lived
- Clear tokens on logout / disconnect

Reference: 09-RESEARCH.md Section 1, 3, 4 for OAuth flow details.
</context>
