<objective>
Implement Notion integration with tools for querying databases, reading/updating pages, and searching content using OAuth-authenticated API calls.
</objective>

<context>
Notion integration is Priority 1 for knowledge management (09-RESEARCH.md). Users want to access their Notion workspace through natural language like "Show me my tasks" or "Add to my notes."

**API Details:**
- Base URL: https://api.notion.com/v1
- Authentication: Bearer token from OAuth 2.0 (integration token)
- Rate limits: ~3 requests per second average
- Scopes: Determined by integration capabilities (workspace-level access)

**Notion API Concepts:**
- **Pages**: Individual content items (can contain text, databases, etc.)
- **Databases**: Structured collections with defined properties (like Tasks, Notes)
- **Blocks**: Content elements within pages (paragraphs, headings, lists, etc.)
- **Properties**: Fields in databases (title, status, date, select, multi-select, etc.)

**Key Endpoints:**
- POST /v1/databases/{database_id}/query - Query database with filters
- GET /v1/databases/{database_id} - Get database metadata (properties)
- GET /v1/pages/{page_id} - Get page metadata
- GET /v1/blocks/{block_id}/children - Get page content (blocks)
- PATCH /v1/pages/{page_id} - Update page properties
- POST /v1/search - Search pages and databases

**Query Filter Syntax:**
Notion queries use a rich filter structure:
```json
{
  "filter": {
    "property": "Status",
    "select": {"equals": "Done"}
  },
  "sorts": [{"property": "Created", "direction": "descending"}]
}
```

**Tool Pattern:**
Follow WebClaw conventions. Notion is unique because:
- Databases have custom schemas (user-defined properties)
- Need to discover database structure before querying
- Rich text requires block-level operations
- OAuth flow includes workspace selection

**Deferred (not in this plan):**
- Creating databases/pages (complex schema definition)
- Block-level editing (append, update blocks)
- Comments and discussions
- User management
- File/media uploads
- Complex rollups and relations querying

         - POST /v1/search

         - Returns matching pages and databases
       
       - `ListDatabases() ([]*Database, error)`
         - Uses Search with filter type=database
         - Convenience method for listing user's databases

       - Notion-Version: "2022-06-28" (required)
       - Content-Type: application/json

       - start_cursor query param (GET) or body field (POST)
       - HasMore indicates more results available
       - Helper to fetch all results for small result sets
       
    5. Error handling:
       - Parse NotionError structure
       - Common errors: invalid_request, object_not_found, rate_limited
       - Rate limit: 429 status, retry after 1 second (3 req/sec average)

  <name>Task 3: Create query filter builder</name>
  <files>internal/integrations/notion/query.go</files>
  <action>
    Build fluent API for constructing Notion database queries.
    
    1. Define Query struct:
       ```go
       type Query struct {
           Filter     *Filter    `json:"filter,omitempty"`
           Sorts      []Sort     `json:"sorts,omitempty"`
           StartCursor string    `json:"start_cursor,omitempty"`
           PageSize   int        `json:"page_size,omitempty"`
       }
       
       type Filter struct {
           // Single property filter
           Property string          `json:"property,omitempty"`
           Title    *TitleFilter    `json:"title,omitempty"`
           RichText *RichTextFilter `json:"rich_text,omitempty"`
           Select   *SelectFilter   `json:"select,omitempty"`
           MultiSelect *MultiSelectFilter `json:"multi_select,omitempty"`
           Date     *DateFilter     `json:"date,omitempty"`
           Checkbox *CheckboxFilter `json:"checkbox,omitempty"`
           Number   *NumberFilter   `json:"number,omitempty"`
           
           // Compound filters
           And []*Filter `json:"and,omitempty"`
           Or  []*Filter `json:"or,omitempty"`
       }
       
       // Filter types (example for select)
       type SelectFilter struct {
           Equals       string `json:"equals,omitempty"`
           DoesNotEqual string `json:"does_not_equal,omitempty"`
           IsEmpty      bool   `json:"is_empty,omitempty"`
           IsNotEmpty   bool   `json:"is_not_empty,omitempty"`
       }
       
       type DateFilter struct {
           Equals     string `json:"equals,omitempty"` // ISO 8601
           Before     string `json:"before,omitempty"`
           After      string `json:"after,omitempty"`
           OnOrBefore string `json:"on_or_before,omitempty"`
           OnOrAfter  string `json:"on_or_after,omitempty"`
           IsEmpty    bool   `json:"is_empty,omitempty"`
           IsNotEmpty bool   `json:"is_not_empty,omitempty"`
           PastWeek   *struct{} `json:"past_week,omitempty"`
           PastMonth  *struct{} `json:"past_month,omitempty"`
       }
       
       type Sort struct {
           Property   string `json:"property"`
           Timestamp  string `json:"timestamp,omitempty"` // created_time, last_edited_time
           Direction  string `json:"direction"` // ascending, descending
       }
       ```
    
    2. Create QueryBuilder:
       ```go
       type QueryBuilder struct {
           query Query
       }
       
       func NewQuery() *QueryBuilder
       
       // Filter builders
       func (qb *QueryBuilder) WhereTitle(property, operator, value string) *QueryBuilder
       func (qb *QueryBuilder) WhereSelect(property, value string) *QueryBuilder
       func (qb *QueryBuilder) WhereDateAfter(property, date string) *QueryBuilder
       func (qb *QueryBuilder) WhereCheckbox(property string, checked bool) *QueryBuilder
       func (qb *QueryBuilder) And(filters ...*Filter) *QueryBuilder
       func (qb *QueryBuilder) Or(filters ...*Filter) *QueryBuilder
       
       // Sort builders
       func (qb *QueryBuilder) OrderBy(property, direction string) *QueryBuilder
       func (qb *QueryBuilder) OrderByCreated(direction string) *QueryBuilder
       
       // Pagination
       func (qb *QueryBuilder) Limit(n int) *QueryBuilder
       
       // Build
       func (qb *QueryBuilder) Build() *Query
       ```
    
    3. Common query patterns:
       - Get tasks with status "Not Started":
         `NewQuery().WhereSelect("Status", "Not Started").Build()`
       - Get items created this week:
         `NewQuery().WhereDateAfter("Created", "2024-01-01").Build()`
       - Get unchecked todos:
         `NewQuery().WhereCheckbox("Done", false).Build()`
       
    4. Query validation:
       - Validate property names exist in database schema
       - Validate filter values against select options
       - Return helpful error if query invalid
