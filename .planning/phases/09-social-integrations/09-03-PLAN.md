<objective>
Implement Google Workspace integration with Gmail (send, read, search) and Calendar (create, view, delete events) tools using OAuth-authenticated API calls.
</objective>

<context>
Google integration is Priority 1 (09-RESEARCH.md) with high user value for productivity. Users want to manage email and calendar through natural language like "Send email to boss" or "What's on my calendar today?"

**API Details:**
- Gmail API: https://gmail.googleapis.com/gmail/v1
- Calendar API: https://www.googleapis.com/calendar/v3
- Authentication: Same OAuth 2.0 token for both APIs
- Scopes needed: https://www.googleapis.com/auth/gmail.modify, https://www.googleapis.com/auth/calendar.events

**Gmail API Endpoints:**
- GET /users/me/messages - List messages
- GET /users/me/messages/{id} - Get message (with format=full for body)
- POST /users/me/messages/send - Send message (RFC 2822 format, base64url encoded)
- GET /users/me/messages?q={query} - Search with Gmail query syntax
- POST /users/me/messages/{id}/trash - Delete/move to trash

**Calendar API Endpoints:**
- GET /calendars/primary/events - List events
- POST /calendars/primary/events - Create event
- GET /calendars/primary/events/{id} - Get event
- DELETE /calendars/primary/events/{id} - Delete event
- GET /calendars/primary/events?timeMin={RFC3339}&timeMax={RFC3339} - Query time range

**Tool Pattern:**
Follow WebClaw tool conventions. Gmail and Calendar are separate namespaces but share OAuth token (both use 'google' provider in OAuthManager).

**Gmail Query Syntax:**
- from:someone@example.com
- to:someone@example.com
- subject:hello
- has:attachment
- is:unread
- after:2024/01/01 before:2024/12/31
- "exact phrase"

**Deferred (not in this plan):**
- Calendar update (PATCH) - less common, can delete+recreate
- Gmail labels/folders management
- Gmail attachments (complex base64 handling, size limits)
- Calendar recurring events
- Calendar attendees/invitations
- Multiple calendars (only primary)

Reference: 09-CONTEXT.md Section "Gmail Use Cases", "Google Calendar Use Cases"
</context>

       type Message struct {
           ID       string `json:"id"`
           ThreadID string `json:"threadId"`

       type MessagePart struct {

           Parts    []*MessagePart `json:"parts"` // For multipart
       }
       
       type MessagePartBody struct {
           Data     string `json:"data"` // base64url encoded
           Size     int    `json:"size"`
       }
       
       type ListMessagesResponse struct {
           Messages []*Message `json:"messages"`
           NextPageToken string `json:"nextPageToken"`
           ResultSizeEstimate int `json:"resultSizeEstimate"`
       }
       ```

       type GmailClient struct {

       }
       
       func NewGmailClient(baseClient *google.Client) *GmailClient
       ```
       
       Implement methods:
       - `ListMessages(maxResults int, query string, pageToken string) (*ListMessagesResponse, error)`
         - GET /users/me/messages
         - Query params: maxResults, q (search query), pageToken
       
       - `GetMessage(id string, format string) (*Message, error)`
         - GET /users/me/messages/{id}
         - Format: "minimal", "full", "raw"
       
       - `SendMessage(to, subject, body string) (*Message, error)`
         - POST /users/me/messages/send

    4. Body extraction helper:
       ```go
       func extractBody(message *Message) string
       ```
       - Navigate message.parts to find text/plain or text/html
       - Decode base64url data
       - Return plain text (strip HTML tags if needed)
